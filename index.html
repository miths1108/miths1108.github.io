Projects

M.E. Course Projects

  • Feature Mining for Similarity Search (Topics in Database Systems, Spring 2015)
    The project's goal is to mine interesting/rare features (set of attributes that makes an object (possesing it) standout). An interesting feature is simply an attributes combination that is present much lower than the expectation. Henceforth, these features can help in drawing similary between objects of interest.

  • Cooperative Game Theory for Information Difussion (Game Theory, Spring 2015)
    In this project, we studied several information diffusion models (independent cascade, linear threshold and decreasing cascade) and simulated the algorithms for influnce maximization on these models. The algorithms implemented in this project were the KKT greedy algorithm and the Shapley value algorithm. Also, this project introduced a new class of information diffusion models (named evolutionary models).

  • Ensemble Learning Methods and Application on Face Detection (Machine Learning, Spring 2015)
    This project gives a comparative study of two popular ensemble techniques, viz. Bagging and AdaBoost. For comparision, decision trees, decision stump, SVM and perceptron were picked as weak learners. Also, we studied the limitations of AdaBoost and explored an improved technique called the FloadBoost algorithm. Lastly, the project includes a small face detection application which uses the Viola-Jones technique (based on AdaBoosting).

  • Automatic Schema Detection from SQL Query Workload (DBMS, Fall 2014)
    This project gives an approach to generate schema from the given set of queries. The generated schema includes information about the table-column mapping, datatype of columns, primary-key foreign-key relationship and dimension table fact table. We have verified the effectiveness of our algorithm by running it on TPC-H query benchmark and comparing the output with the original schema.

  • Activity Analysis in School (Human Computer Interaction, Fall 2013)

Our aim was to design activity systems for different activities that take place in these sections and also explore the ways in which we can automate these activities.

  • Mining Wikipedia Revision History (Minor Project II, Spring 2013)
    This project had two objectives: (i) to model editors' lifetime, and (ii) design a recommendation system for the editors. We used the technique of survival analysis to investigate how long Wikipedia editors remain active in editing. Then the system recommended the editors the top pages on the basis of his domain that he might like to edit. For this, we implemented the algorithm of LDA.
